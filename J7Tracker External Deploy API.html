POST /deploy Deploy Token via HTTPS ▼

Description

Deploy tokens by sending a JSON POST request. Same functionality as WebSocket but via REST API.

Request Body Schema

Name Type Required Description
api_key string ✓ Your wallet API key
type string ✓ Platform: pump, bonk, bags, bnb, nadfun, usd1, clanker
session_id string - JWT session token — links deploy to your J7Tracker account
name string - Token name (max 32 chars)
ticker string - Token symbol/ticker
twitter string - Twitter/X URL
website string - Website URL
buy_amount number - Amount to buy (default: 5)
multi_deploy number 1 Number of tokens to create (1-5, default: 1)
image_type string - Image generation: ascii, sol_ascii, letter
image_data string - Custom image URL or data
ref string - Ref username for fee splitting
gwei number 5 BNB gas price in Gwei (for type: bnb only, default: 5)
bnb_currency string bnb Base currency for Four.Meme BNB deploys: "bnb" or "usd1"
mayhem_mode boolean false Pump.fun only: Enable Token2022 Mayhem Mode
cashback_mode boolean false Pump.fun only: Enable Cashback mode (0% creator fees)
agent_buyback boolean false Pump.fun only: Enable Creator Mode — auto buyback using creator fees (flywheel)
auto_sell boolean false Auto-sell after creation (pump, bonk, usd1, bags)
pump_fee_shareholders array - Pump.fun only: Fee sharing — wallet, GitHub, or Twitter (must total 10000 bps)
bonkers_mode boolean false Bonk/USD1 only: Use letsbonk.fun platform config
sell_panel_enabled boolean - Open sell panel in browser on success (requires session_id)
private_desc boolean false If true, omits the default description from token metadata
vanity_mint_privkey string - Base58 private key for a custom mint address (vanity address). If provided, the token will be created with this keypair as the mint instead of a random or vanity-pool one.
priority_fee_sol number - Priority fee in SOL added to the transaction for faster processing
tip number - Jito tip in SOL — bribes the block builder for guaranteed inclusion

Example Request (BNB with USD1)

POST /deploy Content-Type: application/json { "api_key": "your_encrypted_api_key_here", "type": "bnb", "name": "My Token", "ticker": "MTK", "twitter": "https://twitter.com/mytoken", "website": "https://mytoken.com", "buy_amount": 0.1, "gwei": 5, "bnb_currency": "usd1" }

Responses

200 Deploy Success
{ "type": "token_create_success", "success": true, "message": "Token created successfully", "name": "My Token", "ticker": "MTK", "mode": "pump", "address": "pool_address_here", "mint_address": "token_mint_address_here", "signature": "transaction_signature_here", "request_id": 1 }
400 Deploy Error
{ "type": "token_create_error", "success": false, "error": "Error description here", "request_id": 1 }
400 Validation Error
{ "success": false, "error": "api_key is required and must be a non-empty string" }
503 Service Unavailable
{ "success": false, "error": "Deploy server not connected. Please try again later." }

INFO Bundle Mode Parameters ▼

Bundle Mode

Bundle mode allows multiple wallets to buy in the same transaction block. This ensures all buys happen atomically together with the token creation.

Supported platforms: Pump.fun, Bonk, USD1, Bags

Name Type Default Description
bundle boolean false Enable bundle mode (multiple wallet buys in same block)
bundle_wallets string - Comma-separated wallet entries (see format below)
sniper_wallets string - Comma-separated sniper wallet entries with delay support (see format below). Fires buys after the create tx lands. Works on: Pump.fun, Bonk, USD1, Bags

Bundle Wallets Format

The bundle_wallets string contains comma-separated entries, each with format:

encrypted_api_key:buy_amount_in_sol
  • encrypted_api_key - The encrypted private key of the bundle wallet
  • buy_amount_in_sol - Amount of SOL this wallet will use to buy

Example: 3 wallets buying 0.5, 1.0, and 0.25 SOL each:

"bundle_wallets": "encrypted_key_1:0.5,encrypted_key_2:1.0,encrypted_key_3:0.25"

Example Bundle Request

{ "api_key": "your_encrypted_creator_key", "type": "pump", "name": "Bundle Token", "ticker": "BUNDLE", "buy_amount": 1, "bundle": true, "bundle_wallets": "wallet1_key:0.5,wallet2_key:0.3,wallet3_key:0.2" }

This creates a token with 1 SOL creator buy, plus 3 additional wallet buys of 0.5, 0.3, and 0.2 SOL - all in the same block.

Sniper Wallets Format

The sniper_wallets string contains comma-separated entries, each with format:

encrypted_api_key:buy_amount_in_sol:delay_ms
  • encrypted_api_key - The encrypted private key of the sniper wallet
  • buy_amount_in_sol - Amount of SOL this wallet will use to buy
  • delay_ms - Delay in milliseconds before this wallet fires its buy (0 = immediate, max 60000)

Example: 3 sniper wallets — first buys immediately, second after 500ms, third after 2 seconds:

"sniper_wallets": "encrypted_key_1:0.5:0,encrypted_key_2:1.0:500,encrypted_key_3:0.25:2000"

Example Sniper Wallets Request

{ "api_key": "your_encrypted_creator_key", "type": "pump", "name": "Sniped Token", "ticker": "SNIPE", "buy_amount": 1, "sniper_wallets": "wallet1_key:0.5:0,wallet2_key:0.3:500,wallet3_key:0.2:2000" }

Creates a token with 1 SOL creator buy. After the create tx lands, 3 sniper wallets fire buys — wallet 1 immediately, wallet 2 after 500ms, wallet 3 after 2 seconds.

INFO Pump.fun Mode Parameters ▼

Additional Parameters for Pump.fun

When using type: "pump", you can include these optional parameters:

Name Type Default Description
mayhem_mode boolean false Create Token2022 token with Mayhem Mode enabled
cashback_mode boolean false Enable Cashback mode (0% creator fees)
pump_fee_shareholders array - Array of fee shareholders — wallet, GitHub, or Twitter (see details below)
lock_fee_auth boolean false Permanently revoke fee sharing authority after setting shareholders
agent_buyback boolean false Enable Creator Mode — auto buyback using 100% of creator fees (flywheel)

Fee Shareholders Format

The pump_fee_shareholders array contains objects with:

  • address - Solana wallet address OR
  • github_username - GitHub username (with or without @) - resolved to on-chain PDA OR
  • twitter_username - X / Twitter handle (with or without @) - resolved to on-chain PDA
  • share_bps - Share in basis points (100 bps = 1%)

You can mix all three types in one array.

Important: Total share_bps from all shareholders must equal exactly 10000 (100%). No more, no less.

Example: 3 shareholders at ~33% each = 3334 + 3333 + 3333 = 10000 bps

Example Pump.fun Request with Fee Sharing

{ "api_key": "your_encrypted_api_key_here", "type": "pump", "name": "My Pump Token", "ticker": "PUMP", "buy_amount": 5, "mayhem_mode": true, "cashback_mode": false, "pump_fee_shareholders": [ {"address": "wallet_public_key_1", "share_bps": 3334}, {"github_username": "myGitHubUser", "share_bps": 3333}, {"twitter_username": "myTwitterHandle", "share_bps": 3333} ], "lock_fee_auth": true }

Above example: 3334 + 3333 + 3333 = 10000 bps (100%) - valid! GitHub and Twitter usernames are resolved to PDAs automatically.

INFO Bags Mode Parameters ▼

Additional Parameters for Bags

When using type: "bags", you can include these optional parameters:

Name Type Default Description
bags_fee_claimer string - Twitter handle of fee claimer (without @)
bags_creator_share number 0 Creator's fee share percentage (0-100)
bags_fee_claimer_share number 100 Fee claimer's share percentage (0-100)

Example Bags Request

{ "api_key": "your_encrypted_api_key_here", "type": "bags", "name": "My Bags Token", "ticker": "BAGS", "twitter": "https://twitter.com/mybags", "website": "https://mybags.com", "buy_amount": 5, "bags_fee_claimer": "username", "bags_creator_share": 20, "bags_fee_claimer_share": 80, "image_data": "https://example.com/token-image.png", "ref": "testuser" }
INFO Bonk/USD1 Mode Parameters ▼

Additional Parameters for Bonk/USD1

When using type: "bonk" or type: "usd1", you can include:

Name Type Default Description
bonkers_mode boolean false Use letsbonk.fun platform configuration instead of default

Example Bonkers Mode Request

{ "api_key": "your_encrypted_api_key_here", "type": "bonk", "name": "My Bonkers Token", "ticker": "BONK", "buy_amount": 5, "bonkers_mode": true }

⚠️ Security Notes

  • Never share your API key - It controls your wallet
  • Use HTTPS/WSS - Always use secure connections
  • Validate responses - Check success field before proceeding
  • Rate limiting - Don't spam requests
